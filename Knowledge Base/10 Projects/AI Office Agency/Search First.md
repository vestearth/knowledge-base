# Search First

## Reusable Pattern

Before giving a confident answer, search the live project for the exact term, file, route, task id, or contract.

## Steps

1. Search for the keyword or task id.
2. Open the highest-signal source files.
3. Trace the code, config, docs, or artifacts that produce the behavior.
4. Separate source-backed facts from assumptions.
5. Answer with the evidence path.

## Good Search Targets

- Task ids: `TASK-029`, `TASK-069`
- Workflow phases: `validation_failed`, `assigned_parallel`, `free_roam_complete`
- Contract terms: `manual advisory lane`, `state_model.source_of_truth`
- Dashboard behavior: route names, component names, service names
- Config and profile keys: `dependency_guard`, `loop_guard`, `runner_selector`
